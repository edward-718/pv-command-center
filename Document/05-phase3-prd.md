# PV智枢 第三阶段 PRD

> 版本：v1.0
> 日期：2026-06-30
> 范围：看板视图 + 批量操作 + 高级搜索 + CSV 导入导出 + 动态流
> 前置条件：第一阶段、第二阶段已完成（法规时限、通知中心、标签页框架、MedDRA、随访、提交追踪、任务依赖）

---

## 一、需求背景

前两个阶段已完成：
- **第一阶段**：数据结构扩展、法规时限引擎、通知中心、五标签页框架
- **第二阶段**：MedDRA 医学评估、随访管理、监管提交追踪、任务依赖关系

当前产品已具备完整的 ICSR 处理闭环（时限计算 → 医学评估 → 随访更新 → 监管提交）。但操作效率仍有明显瓶颈：项目经理每天需要逐个分配/调整任务，无法批量处理；任务只有列表视图，缺乏直观的工作流全貌；历史数据无法从 Excel 批量导入；团队动态缺乏可见性。

第三阶段目标：通过看板视图、批量操作、高级搜索和 CSV 导入导出，将产品从「能用」提升到「高效好用」。

---

## 三、用户故事

### 故事 1：项目经理在看板视图上拖拽管理任务

> 作为 PV 项目经理，我需要在看板视图中直观看到所有任务的流转状态，并通过拖拽快速推进任务。

**验收标准**：
- 任务工作台顶部增加「列表 / 看板」视图切换按钮
- 看板视图按任务状态分为 5 列：未开始、处理中、待复核、需补充、已完成
- 每列中任务以卡片形式展示（标题、负责人、截止日、风险标签、法规截止标签）
- 支持拖拽卡片到不同列，触发状态变更（需权限校验）
- 被阻塞的任务在「未开始」列显示阻塞标记，不可拖拽
- 看板列标题显示该状态的任务数量
- 视图切换状态持久化（保存到 localStorage）

### 故事 2：项目经理批量操作多个任务

> 作为 PV 项目经理，我经常需要同时分配多个任务或调整截止日，逐个操作太慢。

**验收标准**：
- 任务列表每行增加复选框（表头有全选/取消全选）
- 选中至少一个任务后，底部浮出批量操作栏
- 批量操作：
  - 批量分配负责人（下拉选择角色，自动分配该角色下的可用人员）
  - 批量调整风险等级（高 / 中 / 低）
  - 批量调整截止日（相对当前日期偏移 N 天）
- 批量操作前弹窗确认，显示影响任务数量和操作类型
- 批量操作记录到审计日志（action: batch_update）
- 操作完成后批量操作栏自动收起

### 故事 3：Processor 使用高级搜索定位任务

> 作为 Case Processor，我需要通过产品、地区、时间范围、严重性等多条件快速找到特定任务。

**验收标准**：
- 搜索栏增加「高级搜索」展开/折叠按钮
- 展开后显示多条件筛选面板：
  - 关键词（标题、病例编号）
  - 项目（多选）
  - 负责人（多选）
  - 截止日范围（开始日 - 结束日）
  - 风险等级（多选）
  - 状态（多选）
  - 严重性（多选）
- 筛选条件实时生效，无需手动提交
- 支持保存当前筛选条件为一个「快捷筛选」（输入名称）
- 快捷筛选以标签形式显示在搜索栏下方，点击一键应用
- 支持删除已保存的快捷筛选

### 故事 4：项目经理导出周报 CSV

> 作为 PV 项目经理，我需要将当前筛选的任务列表导出为 CSV，用于周报汇报。

**验收标准**：
- 任务工作台增加「导出 CSV」按钮
- 导出当前筛选结果（包括搜索和高级搜索条件）
- 导出字段：任务标题、病例编号、项目、产品、地区、负责人、状态、风险、截止日、严重性、MedDRA PT、证据完整度、随访轮次
- 导出文件自动下载，文件名格式：`PV任务导出_YYYY-MM-DD.csv`
- CSV 编码为 UTF-8（含 BOM，兼容 Excel 中文）

### 故事 5：项目经理从 CSV 导入历史任务

> 作为 PV 项目经理，我需要将现有 Excel 中的历史任务批量导入到系统中。

**验收标准**：
- 任务工作台增加「导入 CSV」按钮
- 点击后打开文件选择器，只接受 .csv 文件
- 导入预览弹窗：
  - 显示 CSV 前 5 行数据预览
  - 字段映射：CSV 列名 → 系统字段（可手动选择）
  - 预计导入任务数
  - 错误行高亮提示
- 导入规则：
  - 标题为必填字段，缺失则该行跳过并记录到导入报告
  - 截止日格式兼容 YYYY-MM-DD 和 YYYY/MM/DD
  - 已存在的任务标题（模糊匹配）提示是否覆盖
- 导入完成后显示摘要：成功 N 条、跳过 N 条、失败 N 条
- 导入操作记录到审计日志

### 故事 6：团队成员在驾驶舱查看最近活动

> 作为 PV 项目经理，我需要在驾驶舱一眼看到团队最近在做什么。

**验收标准**：
- 驾驶舱右侧（桌面端）或下方（移动端）增加「最近活动」面板
- 展示最近 20 条团队活动记录
- 活动类型：任务状态变更、评论添加、证据上传、复核通过/退回、随访创建、提交记录添加、项目创建
- 每条记录：操作人角色、操作描述、目标对象名称、时间
- 点击目标对象直接跳转到对应任务详情
- 活动记录从 state.activities 读取，按时间倒序

## 五、业务规则

### 5.1 看板视图拖拽规则

| 规则 | 说明 |
|---|---|
| 拖拽方向 | 只能从左侧列拖到右侧列（如「未开始」→「处理中」），不允许反向拖拽（如「已完成」→「待复核」） |
| 权限校验 | 拖拽到「待复核」列时，需要当前角色有复核权限，否则提示无权限并回退卡片 |
| 阻塞任务 | 被阻塞的任务显示「阻塞」水印，不允许拖拽到「处理中」或后续列 |
| 阻塞列 | 看板增加「阻塞」区域，被阻塞的任务自动归入，不占用状态列 |
| 状态注释 | 拖拽到「待复核」和「需补充」列时，弹窗要求填写说明（复用现有状态变更逻辑） |
| 任务数量 | 看板列标题显示任务数量，列内无任务时显示空状态提示 |

### 5.2 批量操作规则

| 规则 | 说明 |
|---|---|
| 选择范围 | 批量操作仅影响当前筛选结果中的已选中任务 |
| 全选 | 表头全选仅选中当前筛选结果（不是全部任务） |
| 权限校验 | 批量分配负责人时，不能分配给比自己权限更高的角色（Vendor 不能批量分配给 PM） |
| 截止日调整 | 仅允许向后延期（不缩短截止日），防止误操作导致任务超期 |
| 批量状态变更 | 批量操作不包含批量状态变更（状态变更涉及复核流程，需要逐个处理） |
| 审计记录 | 每次批量操作生成一条审计日志，包含操作类型、影响任务数和操作详情 |

### 5.3 CSV 导入规则

| 规则 | 说明 |
|---|---|
| 文件编码 | UTF-8（含 BOM）或 GBK |
| 必填字段 | title（标题） |
| 可选字段 | assigneeRole, dueDate, risk, projectId, description |
| 截止日格式 | YYYY-MM-DD、YYYY/MM/DD、YYYY年MM月DD日 |
| 重复检测 | 如果已存在相同标题的任务，提示用户选择跳过或覆盖 |
| 数量限制 | 单次导入上限 100 条 |
| 随访任务 | CSV 导入的任务 followUpRound 默认为 0（初报任务） |

### 5.4 CSV 导出规则

| 规则 | 说明 |
|---|---|
| 导出范围 | 当前筛选结果（包括搜索和高级搜索条件） |
| 导出字段 | 任务标题、病例编号、项目名称、产品、地区、负责人、状态、风险、截止日、严重性、MedDRA PT、证据完整度、随访轮次 |
| 文件编码 | UTF-8 with BOM（兼容 Excel 中文） |
| 日期格式 | YYYY-MM-DD |
| 空值处理 | 空字段导出为空字符串 |

## 七、新增函数清单

| 函数 | 功能 | 调用时机 |
|---|---|---|
| `renderKanban()` | 渲染看板视图 | 用户切换到看板视图时 |
| `initKanbanDragDrop()` | 初始化拖拽交互 | 看板视图渲染后 |
| `handleKanbanDrop(taskId, newStatus)` | 处理看板拖拽状态变更 | 拖拽放置时 |
| `renderBatchToolbar(selectedTasks)` | 渲染批量操作浮动栏 | 选中任务后 |
| `executeBatchAction(action, params)` | 执行批量操作 | 用户确认批量操作后 |
| `toggleAdvancedSearch()` | 展开/折叠高级搜索面板 | 用户点击按钮时 |
| `renderAdvancedSearchFilters()` | 渲染高级搜索筛选条件 | 面板展开时 |
| `applyFilters()` | 应用所有筛选条件（搜索+高级） | 筛选条件变更时 |
| `saveCurrentFilter(name)` | 保存当前筛选为快捷筛选 | 用户点击保存时 |
| `deleteSavedFilter(id)` | 删除已保存的快捷筛选 | 用户点击删除时 |
| `renderQuickFilters()` | 渲染快捷筛选标签栏 | 筛选条件变更时 |
| `exportCSV()` | 导出当前筛选结果为 CSV | 用户点击导出按钮时 |
| `openCSVImportDialog()` | 打开 CSV 导入弹窗 | 用户点击导入按钮时 |
| `parseCSVFile(file)` | 解析 CSV 文件并预览 | 文件选择后 |
| `executeCSVImport(mapping)` | 执行 CSV 导入 | 用户确认导入后 |
| `renderActivitiesPanel()` | 渲染驾驶舱最近活动面板 | 渲染驾驶舱时 |

## 十、风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| HTML5 拖拽兼容性问题 | 高 | 拖拽使用 HTML5 Drag & Drop API，同时提供点击状态变更作为备选操作 |
| 批量操作误操作 | 中 | 弹窗确认显示影响范围；不提供批量删除和批量状态变更功能 |
| CSV 导入数据质量差 | 中 | 导入预览 + 错误行高亮 + 导入摘要；单次导入上限 100 条 |
| 高级搜索条件过多导致 UI 拥挤 | 中 | 默认折叠，点击展开；筛选条件分组排列 |
| 动态流数据量过大 | 低 | 只展示最近 20 条，不加载全部历史 |
| 看板列内容过多滚动困难 | 低 | 看板列设置 max-height 和 overflow-y: auto |
