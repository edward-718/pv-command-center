# PV智枢 第一阶段 PRD

> 版本：v1.0
> 日期：2026-06-29
> 范围：数据结构扩展 + 法规时限引擎 + 通知中心 + 任务详情标签页重构

---

## 一、需求背景

MVP 已实现 PV 项目管理的通用流程（项目创建、任务分配、状态流转、审计日志）。但缺少 PV 行业最核心的「法规时限管理」能力——PV 从业者每天需要计算从 Day 0 到法定提交日的剩余时间，并确保在截止前完成医学审评、质量复核和监管提交。

第一阶段目标：让产品具备「智能时限计算 + 主动提醒」的能力，同时重构任务详情界面以支撑后续医学字段的扩展。

## 三、用户故事

### 故事 1：项目经理创建带法规时限的 ICSR 项目

> 作为 PV 项目经理，我需要创建一个 ICSR 项目并指定 Day 0（获知日期），以便系统自动计算法定提交截止日。

**验收标准**：
- 创建项目弹窗中，ICSR 类型项目显示「Day 0」日期选择器和「法规规则」下拉框
- 法规规则选项：NMPA-15d、NMPA-非严重-30d、EMA-15d、FDA-15d、custom
- 选择 Day 0 和法规规则后，系统自动计算出法定截止日并显示在弹窗中
- 创建项目后，项目卡片显示「法规截止：YYYY-MM-DD」

### 故事 2：系统自动生成时限提醒

> 作为 Case Processor，我需要在任务临近截止时收到提醒，避免遗漏监管时限。

**验收标准**：
- 任务截止前 24 小时，系统生成「即将逾期」通知
- 任务截止前 4 小时，系统生成「紧急：即将逾期」通知
- 任务逾期后，系统生成「已逾期」通知
- 任务被复核退回时，系统生成「复核退回」通知
- 通知出现在通知中心，并在导航栏铃铛图标上显示未读数量角标

### 故事 3：用户在通知中心查看和处理通知

> 作为 PV 项目经理，我需要在一个地方查看所有系统通知，并快速跳转到对应任务。

**验收标准**：
- 点击导航栏铃铛图标，展开通知下拉面板
- 面板显示最近 10 条通知（按时间倒序）
- 通知显示：类型图标、标题、时间、关联任务/项目
- 点击通知项，直接跳转到对应任务详情弹窗
- 支持标记单条通知为已读，或一键全部已读
- 未读通知在导航栏显示红色角标数字

### 故事 4：任务详情展示多标签页

> 作为 Safety Physician，我需要在任务详情中查看不同维度的信息（评论、随访、医学评估、监管提交、证据）。

**验收标准**：
- 任务详情弹窗顶部显示 5 个标签：评论与结论、随访、医学评估、监管提交、证据与日志
- 点击标签切换内容区域，切换流畅无闪烁
- 「评论与结论」标签保留现有评论列表和新增评论功能
- 「随访」「医学评估」「监管提交」标签在本阶段显示占位提示（功能在第二批实现）
- 「证据与日志」标签展示附件列表和审计日志（该任务相关）

## 五、业务规则

### 5.1 法规时限计算规则

| 法规规则 | Day 0 偏移天数 | 适用场景 |
|---|---|---|
| NMPA-15d | 15 日历日 | 中国境内严重不良反应 |
| NMPA-非严重-30d | 30 日历日 | 中国境内非严重不良反应 |
| EMA-15d | 15 日历日 | 欧盟严重不良反应 |
| FDA-15d | 15 日历日 | 美国严重不良反应 |
| custom | 用户自定义 | 特殊场景 |

**计算方式**：截止日 = Day 0 + 偏移天数（日历日，非工作日）

### 5.2 通知生成规则

| 触发条件 | 通知类型 | 生成时机 | 目标角色 |
|---|---|---|---|
| 距离截止日 ≤ 24h 且 > 4h | due_soon | 实时检测 | 任务负责人 + PM |
| 距离截止日 ≤ 4h 且 > 0 | due_soon_urgent | 实时检测 | 任务负责人 + PM |
| 当前日期 > 截止日 | overdue | 实时检测 | 任务负责人 + PM + QA |
| 任务状态变为「需补充」 | review_returned | 状态变更时 | 任务负责人 |
| 任务缺少必填证据且截止日 < 24h | evidence_missing | 实时检测 | 任务负责人 |

**去重规则**：同一任务同一类型的通知，24 小时内只生成一次。

### 5.3 数据兼容规则

- 旧数据（本阶段前创建的项目和任务）缺少新字段时，用默认值填充
- 旧任务没有 `regulatoryDeadline` 时，显示「未设置」
- 旧项目没有 `dayZero` 时，法规时限功能对该项目不生效

## 七、API 接口（前端模拟）

本阶段为纯前端实现，所有逻辑在 `app.js` 中完成。

### 7.1 新增函数清单

| 函数 | 功能 | 调用时机 |
|---|---|---|
| `calculateRegulatoryDeadline(dayZero, rule)` | 计算法规截止日 | 创建项目时、Day 0 变更时 |
| `getDaysUntilDeadline(deadline)` | 计算距离截止日剩余天数 | 渲染任务列表时 |
| `generateNotifications()` | 扫描所有任务生成通知 | 页面加载时、状态变更后 |
| `addNotification(notification)` | 写入通知并更新角标 | 生成通知时 |
| `markNotificationRead(id)` | 标记单条通知已读 | 用户点击通知时 |
| `markAllNotificationsRead()` | 标记全部已读 | 用户点击「全部已读」时 |
| `renderNotifications()` | 渲染通知列表页面 | 切换到通知中心视图时 |
| `renderNotificationBadge()` | 渲染导航栏角标 | 通知变更时 |
| `renderTaskDetailComments(task)` | 渲染评论标签 | 打开任务详情时 |
| `renderTaskDetailFollowUp(task)` | 渲染随访占位标签 | 打开任务详情时 |
| `renderTaskDetailMedical(task)` | 渲染医学评估占位标签 | 打开任务详情时 |
| `renderTaskDetailSubmission(task)` | 渲染监管提交占位标签 | 打开任务详情时 |
| `renderTaskDetailEvidence(task)` | 渲染证据与日志标签 | 打开任务详情时 |
| `switchTaskTab(tabName)` | 切换任务详情标签 | 用户点击标签时 |
| `addActivity(activity)` | 写入活动记录 | 关键操作后 |
| `renderActivities()` | 渲染动态流 | 渲染驾驶舱时 |

## 九、风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 任务详情重构导致评论功能损坏 | 高 | 保留原有评论 DOM class 和事件监听，只做容器迁移；重构完成后单独测试评论功能 |
| 法规时限计算错误（日历日 vs 工作日） | 高 | 明确使用日历日计算；在 UI 中标注「日历日」避免误解 |
| 通知生成频率过高（刷屏） | 中 | 实现去重规则（同一任务同一类型 24h 内只通知一次）；通知数量上限控制 |
| 旧数据兼容性问题 | 中 | loadState() 中使用默认值填充；恢复演示数据后全面测试 |
| 标签页内容过多导致弹窗过高 | 低 | 标签页内容区域设置 max-height 和 overflow-y: auto |
